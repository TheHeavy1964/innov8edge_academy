## The Four Types of AI Evals

### 1. Correctness Evals
**Question:** Is the answer factually right?
- Compare AI output to verified ground truth answers
- Use exact match, semantic similarity, or expert review scoring
- Best for: RAG systems, Q&A bots, data extraction

### 2. Quality Evals
**Question:** Is the answer good, even if there's no single "right" answer?
- Score on dimensions: helpfulness, tone, completeness, safety
- Use LLM-as-judge or human evaluators on a sample
- Best for: Customer service agents, content generation, complex reasoning

### 3. Regression Evals
**Question:** Did anything break?
- Run on every model/prompt/data change
- Alert if scores drop below threshold vs. baseline
- Best for: Any production AI system with established baselines

### 4. A/B Comparison Evals
**Question:** Which version is better?
- Run two versions against the same test set
- Statistical significance testing on results
- Best for: Prompt optimization, model selection, feature experiments

## Building Your Eval Pipeline

```
1. Create golden dataset (100-500 verified examples)
2. Define scoring rubrics (exact match, LLM-judge, human review)
3. Set baseline scores for current production version
4. Automate eval runs on every change (CI/CD integration)
5. Set alerting thresholds (e.g., alert if accuracy drops >5%)
6. Review eval results before every production promotion
```
