## Architecture Patterns for Scaled AI

### Pattern 1: The Hub-and-Spoke
One orchestrator agent coordinates multiple specialist agents. The orchestrator receives tasks, decomposes them, and delegates to specialists (researcher, writer, reviewer, publisher).
- **Best for:** Complex workflows with distinct, predictable subtasks
- **Risk:** Single point of failure at the orchestrator

### Pattern 2: The Pipeline
Agents process tasks sequentially — each agent transforms the output of the previous.
- **Best for:** Content production, data processing, document transformation
- **Risk:** Error propagation — a bad output in step 2 corrupts all downstream steps

### Pattern 3: The Peer Network
Agents collaborate without a central orchestrator, using shared state and messaging.
- **Best for:** Research tasks, adversarial review (agent A writes, agent B critiques)
- **Risk:** Coordination complexity, potential for contradiction

## Cost Optimization Playbook

| Strategy | Cost Reduction | Complexity |
|---------|---------------|-----------|
| Model routing (GPT-4 → mini for simple tasks) | 60-90% | Low |
| Semantic caching | 30-60% | Medium |
| Prompt compression | 20-40% | Low |
| Async/batch for non-urgent tasks | 30-50% | Medium |
| Context pruning (remove old conversation) | 20-35% | Low |
| Token budgets per request | Prevents spikes | Low |
