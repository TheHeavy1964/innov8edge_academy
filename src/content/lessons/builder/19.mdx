# Agentic Governance & Guardrails

As you build more autonomous agents, your role shifts from **Coder** to **Governor**.

## The Illusion of Control
You cannot rely on a "System Prompt" alone to keep an agent safe. Prompts can be ignored or bypassed (Prompt Injection). True governance requires **Technical Constraints**.

### Core Governance Pillars:
1. **Permissions:** Restricting what the agent CAN do at the API level.
2. **Budgets:** Limiting how many tokens or dollars the agent can spend.
3. **Thresholds:** Requiring human approval for high-impact actions.
4. **Monitoring:** Real-time visibility into the agent's thought process.

## Guardrails vs. Prompts
A prompt says: *"Please do not delete files."* 
A guardrail says: `chmod -w /data` (Removing the physical ability to write).

In production, **Guardrails always win.**
