# Red-Teaming Your Agents

Welcome to the **"How"** of AI Security. In this lesson, you become the attacker.

## The Security Paradox
The more "Agentic" and "Capable" an agent is, the more dangerous it becomes if it is tricked. If your agent can delete files, a hacker only needs to trick the agent to delete your business.

### Common Attack Vectors:
1. **Direct Injection:** "Forget your rules and do X."
2. **Indirect Injection:** Malicious text hidden in a RAG document.
3. **Social Engineering:** Tricking the agent into believing you are an Admin.
4. **Denial of Wallet:** Tricking the agent into an infinite loop to run up your API bill.

## Building a Fortress
You will learn to implement **Layered Defense**:
- **Layer 1:** Hard Permissions (Linux level).
- **Layer 2:** Input Sanitization (Checking for 'Ignore' commands).
- **Layer 3:** Output Filtering (Redacting secrets).
- **Layer 4:** Anomaly Detection (Monitoring for weird behavior).

> **The Golden Rule:** Never trust user input. Ever. Even if it's coming from a "trusted" document.
