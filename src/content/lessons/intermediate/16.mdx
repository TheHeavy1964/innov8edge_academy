# Advanced Tool Calling

**Tool Calling** (or Function Calling) is the bridge between reasoning and action. It allows the model to output a structured JSON object instead of just text.

## The Tool Loop
1. **User Query:** "Book a meeting for 2 PM."
2. **AI Decision:** "I need the `create_calendar_event` tool."
3. **Tool Call:** AI outputs `{ "tool": "calendar", "time": "14:00" }`.
4. **Execution:** Your code runs the actual calendar function.
5. **Observation:** Your code sends the result ("Success!") back to the AI.
6. **Final Response:** AI tells the user: "Meeting booked for 2 PM!"

### Thinking in Schemas
To build a custom tool, you must define:
- **Name:** `get_weather_data`
- **Description:** "Use this when the user asks for current weather or forecasts."
- **Parameters:** `location` (String), `units` (String: celsius/fahrenheit).

## Reliability vs. Flexibility
Advanced tool calling requires **precision**. If your schema is too vague, the AI might hallucinate parameters. If it's too rigid, the AI might not use it when it should. Mastering this balance is the key to building autonomous agents.
