# Advanced Tool Calling: Giving AI Hands

**Tool Calling** is the breakthrough that turned AI from a conversationalist into a collaborator. It allows the model to interact with the real world through your code.

## The Tool Loop

1. **The Model Decides:** The AI looks at the user request and realizes it can't answer without help (e.g., 'What is the stock price of AAPL?').
2. **The Request:** The AI outputs a specific command (e.g., `get_stock_price(symbol="AAPL")`).
3. **The Execution:** Your application sees this, runs the real function, and gets the price ($230.15).
4. **The Completion:** Your app sends the price back to the AI. The AI then tells the user: 'The current price of AAPL is $230.15.'

### Best Practices for Tool Design:
- **Clear Descriptions:** The AI doesn't know what `fn_42()` does. Call it `calculate_mortgage_rate()` and give it a detailed description.
- **Strict Schemas:** Use JSON Schema to ensure the AI provides the right data types (e.g., ensuring a ZIP code is a string, not a number).
- **Safety First:** Always use **Read-only** tools by default. Require human approval for any "Write" actions.

> **Key Takeaway:** A tool is only as good as its description. Treat your tool definitions as "High-Density Instructions" for the AI.
