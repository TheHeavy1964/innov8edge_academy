# Debugging & Error Handling

AI is **Probabilistic**, meaning it won't always give the same output twice. To build production-ready systems, you must design for **Resilience**.

## Common AI Failure Modes
1. **Hallucination:** The AI makes up a tool parameter that doesn't exist.
2. **Schema Break:** The AI outputs text instead of valid JSON.
3. **Context Overflow:** The conversation gets too long and the AI "forgets" the instructions.
4. **Tool Rejection:** The AI refuses to use a tool it thinks is "unsafe."

### The Resilient Architect's Toolkit:
- **Validation Layers:** Use standard code (like Zod or JSON.parse) to verify the AI's output before executing it.
- **Self-Correction:** If the AI fails, send the error back to it: *"Your previous output was invalid JSON. Please try again."*
- **Fallbacks:** If the AI fails twice, route the task to a **Human-in-the-Loop**.

## Error Handling is the Product
In AI engineering, anyone can build a "Happy Path" prototype. A professional is someone who has built the "Error Paths"—the logic that ensures the system stays stable even when the model gets confused.
