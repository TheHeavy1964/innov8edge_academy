# MCP Explained

The **Model Context Protocol (MCP)** is an open standard that enables developers to build secure, two-way integrations between AI models and their local data.

## Why MCP?
Before MCP, if you wanted an AI to read your files, you had to write custom code for that specific AI. If you switched models, you had to start over. MCP standardizes this.

### The Architecture:
1. **The Server:** Connects to your data (e.g., Google Drive, GitHub, SQL).
2. **The Client:** The AI model (e.g., Claude Desktop, your custom app).
3. **The Protocol:** A JSON-RPC based bridge that handles discovery and execution.

## Seeing the Bridge
In our sandbox, you witnessed the "Handshake." This is where the AI asks the MCP Server: *"What can you do?"* and the server responds with a list of available tools. This discovery layer is what makes AI automation truly flexible.
