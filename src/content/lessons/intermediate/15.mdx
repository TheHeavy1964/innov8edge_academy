# Data Pipeline Ethics

RAG is powerful, but it comes with a **Privacy Responsibility**. If you automate the "Retrieval" of data, you must also automate the "Protection" of that data.

## The Sanitization Layer
Before a document reaches the Vector Store, it must go through a **Sanitization Pipeline**.
- **Redaction:** Stripping out SSNs, Emails, and Phone Numbers.
- **De-identification:** Replacing specific names with generic placeholders (e.g., "John Doe" becomes "Customer_A").
- **Classification:** Tagging data as "Internal," "Confidential," or "Public."

### Why it Matters
If an employee asks the AI: *"What is the company's salary structure?"*, you want the AI to retrieve the *policy*, not the *actual spreadsheet* of everyone's pay. 

## Context Ethics
RAG should only retrieve what the user is authorized to see. This is called **Attribute-Based Access Control (ABAC)**. If the user doesn't have "Manager" clearance, the RAG engine shouldn't even search the "HR_Files" folder. 

Protecting your data at the source is the only way to build a production-ready AI system.
